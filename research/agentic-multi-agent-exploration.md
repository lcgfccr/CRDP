# Multi-agent debate, ToT, and exploration patterns — mapping to vault max-ROI

## SOTA survey

### 1. Tree of Thoughts (Yao et al. 2023, NeurIPS)

**Core mechanism.** Generalizes Chain-of-Thought into a search tree. Generate multiple candidate "thoughts" per step, self-evaluate each, BFS/DFS the tree, prune low-value branches, backtrack when stuck. Three components: thought decomposition, generator, evaluator (value function or vote).

**Strengths for research.** Strong on tasks with crisp success criteria (Game of 24: 4% CoT to 74% ToT; Mini Crosswords; Creative Writing). The branching exposes alternative reasoning paths a single chain would miss. Works because evaluator can score partial states.

**Weaknesses for research tasks.** Open-ended research lacks the verifier ToT depends on. There is no "value(node)" for "is this a good claim about OAuth2?" — you would need a separate critic agent, which collapses into multi-agent debate. ToT shines when partial states are scoreable; vault research is largely not scoreable until synthesis.

**Cost.** Hundreds of model calls per task in original work. Branching factor 3-5, depth 3-5 → 3^3 to 5^5 nodes considered, with eval calls per node. Much cheaper if generator and evaluator use different model tiers (paper showed GPT-4 generate + GPT-3.5 eval reaches 64%).

**Failure modes.** Evaluator hallucination poisons pruning. Wrong early-prune kills correct lineage. Verbose deliberation drowns weak signal. Cost balloons fast on real tasks.

### 2. Multi-agent debate (Du et al. 2023; ICLR 2025 critical re-eval)

**Core mechanism.** N agents independently answer, then exchange answers, critique each other's reasoning, revise. Repeat for R rounds. Final answer by majority or convergence.

**Strengths.** Original Du paper: improves factuality, reduces hallucinations on reasoning benchmarks. Cross-checking catches some single-agent failure modes.

**Weaknesses (the 2025 critical view, ICLR Blogposts).** Most MAD frameworks fail to consistently outperform Self-Consistency (resampling a single agent N times). MAD often "fails to preserve correct answers" — agents persuade each other into wrong answers more than CoT does. Multi-Persona variants explicitly forced into opposition prevent recovery. Adding rounds or agents does NOT scale accuracy. Most benchmark questions need single knowledge points → MAD degrades to inefficient resampling.

**Cost.** ~3-5x single-agent for 3 rounds, 3 agents. Up to 26x in worst-case enterprise traces. S2-MAD (NAACL 2025) cuts ~94.5% via redundancy filtering. GroupDebate cuts ~30-46%.

**When MAD actually helps.** Heterogeneous models (e.g., GPT-4o-mini + Llama-3.1-70B beats either alone). Genuinely complex reasoning beyond standard benchmarks. Safety-critical multi-perspective vetting.

**Failure modes.** Sycophantic convergence (correct minority capitulates). Architectural opposition without grounding. Token cost without accuracy gain. Over-aggressive correction.

### 3. AutoSurvey / Agent Laboratory / AI Scientist v2

**Agent Laboratory (Schmidgall 2025).** Three sequential phases (LitReview, Experimentation, Report). Specialized agents: PhD/Postdoc/MLEngineer/Professor roles. mle-solver iterates code via REPLACE/EDIT. Cost: $2.33-$13.10 per workflow. Caveat: autonomously generated papers score 3.5-4.0/10 vs 5.9/10 NeurIPS bar — execution is fine, originality is weak.

**AI Scientist v2 (Sakana 2025).** Removes templates from v1. Uses **agentic best-first tree search** managed by an experiment-manager agent. Parameters: `num_workers` (parallel paths), `steps` (max nodes), `max_debug_depth`. Multiple independent solution trees in Stage 1 (`num_drafts`). Probabilistic debug-vs-abandon for failing paths. ~$15-20 per main experiment. Honest finding: v2 doesn't beat v1 when v1 has a strong template — exploration vs consistency trade-off is real.

**AgentRxiv (2025).** Three Agent Laboratories run in parallel, share preprints via similarity-search server. MATH-500: 70.2% → 79.8% with parallel + shared knowledge; reaches 76.2% in 7 papers parallel vs 23 sequential. Per-paper $3.11, ~1.36 hr. The key compounding mechanism: agents see each other's results asynchronously, build on them.

**Deep Research agents (Aug 2025 survey, arxiv 2508.12752).** Four-stage pipeline: planning → question developing → web exploration → report generation. Multi-agent variants (planner / query / retriever / writer specialists) outperform monolithic when components need different optimization. Honest limit: "plans generated by current LLMs often lack robustness to ambiguous research questions" — planning decomposition is the bottleneck.

**Enterprise Deep Research.** Maintains progressively refined knowledge representation across iterations. Each iteration starts with original objective + distilled prior summary, NOT raw accumulated context. Prevents exponential context growth.

### 4. Self-directed exploration / curiosity-driven

**SelfAI (arxiv 2512.00403, 2025).** Trajectory-driven decision-making for long-horizon discovery. Reasons over accumulated experimental trajectories to pick next action. Adaptive stopping for unproductive paths. Explicit efficiency-diversity trade-off knob. Targets the failure mode of vanilla agents that don't learn from their own search history.

**Deep Ideation.** LLM iteratively explores keyword relationships in scientific concept network, expanding seeds based on expert evaluation. Each cycle's keywords feed next cycle's ideation. Compounding via expanded conceptual graph.

**DORA Explorer.** Training-free exploration improvement at inference time. Targets the "exploitation lock" failure where agents stop trying new directions once any direction works.

**FMSP (Foundation Model Self-Play).** Diverse populations of competing agents with foundation-model search. Open-ended discovery via population dynamics rather than fixed prompts. Heavy compute.

### 5. Iterative deepening / progressive elaboration

**Standard pattern.** Initial broad pass → identify gaps → targeted deep pass → re-evaluate → broad sweep for emergent gaps. The vault's `/vault-autoresearch` 3-round (orient → deepen → synthesize) is a textbook instance.

**ResearchAgent (2024).** Iterative idea generation over scientific lit. Each round refines on prior + literature feedback.

**IMPROVE (2025).** Iterative model pipeline refinement — adjust components per empirical feedback, not whole-pipeline rewrites.

**Common loop shape.** generate → validate → diagnose → targeted refinement → repeat. Targeted refinement (only the failing segment) is the cost-saving move vs whole-output regeneration.

### 6. Society of Mind in 2024-2025

Minsky's framing keeps showing up but mostly as branding. Hierarchical multi-agent (planner-coordinator-workers) outperforms flat MAD on strategic reasoning tasks (RTS games, Sibyl). The substantive insight: heterogeneity + role specialization beats N copies of the same agent debating. Pure debate among identical agents is the failure mode.

## Mapping to vault

### Where vault already covers SOTA

**`/vault-landscape` (5-persona parallel).** Already implements heterogeneous-perspective parallel exploration — exactly the pattern that beats homogeneous MAD. Personas (landscape/mechanics/failure-modes/stakeholders/adversarial) are role-specialized agents writing to filesystem, merged via 4-pass synthesis. This is closer to Society of Mind / hierarchical multi-agent than to pure debate. Adding Du-style debate ON TOP would burn tokens for marginal gain — the personas already disagree productively, and the merge step surfaces tensions verbatim instead of forcing convergence (which is what kills MAD).

**`/vault-autoresearch` (3-round).** Textbook iterative deepening (orient → deepen → synthesize). Round-3 mandatory counter-evidence pass + optional `--challenge` round 4 = adversarial verifier in the loop. Matches Enterprise Deep Research's progressively-refined-knowledge pattern.

**`/vault-challenge`.** Standalone adversarial falsification page-by-page. Closes the Popper gap MAD claims to fill but actually causes (sycophantic convergence). Vault does this AFTER synthesis, not during, which is more honest.

**`/vault-hypothesize` + `/vault-probe`.** Self-directed agenda generation. Vault produces its own research questions from existing pages. Matches SelfAI's intent-translation but bottoms-up from accumulated content rather than top-down from research goal.

**`/vault-synthesize` + `/vault-analogize`.** Cross-domain recombination. Analogize discovers candidate juxtapositions by structural pattern. This is the move ToT cannot make (no verifier for "good analogy").

**Conclusion on coverage.** Vault has the multi-perspective layer, the iterative-deepening layer, the adversarial layer, and the self-direction layer. The architectural gaps are in *adaptive routing*, not in *adding more agents*.

### What the vault is missing

**1. Adaptive depth — branch-on-tension.** `/vault-autoresearch` is fixed 3 rounds (or 1/5 via flags). It does not branch when round 2 surfaces a contradiction worth chasing. SelfAI's trajectory-driven stopping/branching is the missing piece. If round 2 finds claim X conflicts with claim Y, current behavior is to log the tension in `## Tensions & contradictions` and move on. A ToT-flavored branch would spawn a sub-research on the tension itself.

**2. Topic-aware quality policy.** User's actual ask. Currently every autoresearch hits 3 rounds, every landscape hits 5 personas. A topic like "JWT exp claim" is over-served at 3 rounds; "agentic frameworks for quant trading" may be under-served. Need a planner agent that reads the topic + existing vault state and emits a *policy* (rounds, personas, challenge-required, depth-budget) before research starts. This is one agent, not a debate.

**3. Cross-page coherence as a search signal.** Vault has `/vault-lint` (health check) but not a *gap-driven explorer* that picks the next research target by analyzing where the vault graph is incoherent (orphan claims, contradicted claims, weakly-cited load-bearing claims). `/vault-probe` finds blind spots the user did not articulate; it does not yet drive autonomous follow-up.

**4. Compounding via parallel runs.** AgentRxiv shows 70.2% → 79.8% with three parallel labs sharing a preprint store. Vault is single-user, single-thread; there is no "parallel research" pattern except `/vault-landscape`'s one-shot fan-out. Probably out of scope unless the user wants to run multiple `/vault-autoresearch` jobs concurrently with shared raw/.

### Direct answers to the user's framing questions

**Q1: What does multi-agent debate add beyond `/vault-landscape`?**
Almost nothing useful. Landscape already does heterogeneous parallel agents with a merge step that preserves disagreement. The 2025 critical re-eval of MAD (ICLR) shows debate-style convergence often DELETES correct minority answers. Landscape's "surface tensions verbatim, do not reconcile" is the better pattern. Adding a debate round would risk regression.

**Q2: Should `/vault-autoresearch` adopt ToT-style branching when claims fork?**
Yes — but narrowly. Pure ToT is wrong for research (no verifier). What works: when round 2 surfaces a load-bearing tension, spawn a sub-research on just that tension before round 3 synthesis. Cost: +1 conditional round. Verifier: presence of tension itself. Bounded branching factor (max 1 sub-spawn per autoresearch run), so cost doesn't explode. This is closer to selective deepening than to ToT.

**Q3: Is topic-aware quality determination its own agent?**
Yes. This is the single highest-ROI addition. A "policy generator" agent runs once before research, reads topic + existing vault state (index.md, overview.md, related pages), and emits a structured policy: `rounds: 2|3|5`, `personas: [list]`, `challenge: bool`, `branch_on_tension: bool`, `verification: quick|deep`. Downstream skills consume the policy. Default to current behavior on policy failure. Adds ~1 model call per research run, saves on over-served tasks, raises ceiling on under-served ones.

**Q4: Should vault decide what to research NEXT autonomously?**
Partially yes. Build on existing scaffolding. `/vault-hypothesize` and `/vault-probe` already produce candidate next-targets. Missing: an autopilot that picks one and executes without user prompting. Risk: runaway WebFetch budget. Right answer: a `/vault-next` (or extension to `/vault-autoresearch` with `--auto-pick`) that runs probe → leverage-rank candidates → present top 3 with cost estimates → user picks (or `--no-confirm` for true autonomy). Compose existing primitives, do not build a new exploration framework from scratch.

**Q5: Concrete proposals — adjacent capability, not duplicate.**

**Proposal A: `/vault-policy` — topic-aware quality planner.**
- Single-agent, one model call before research.
- Inputs: topic, project slug, optional flags.
- Reads: index.md, overview.md, pages/*/frontmatter (titles + tags), questions.md.
- Outputs: policy struct consumed by `/vault-autoresearch` and `/vault-landscape`.
- Decision factors: topic specificity (narrow vs broad), vault prior coverage (none / partial / heavy), user-stated stakes, presence of contradicting prior claims.
- Cheap. Composable. Matches user's "system determines its own quality standards per topic."
- Failure mode: bad policy → degenerate to default. Always show policy to user before spending real budget.

**Proposal B: `/vault-autoresearch --branch-on-tension` flag.**
- Round 2 detects tension between sources OR between new findings and existing vault page.
- If tension is load-bearing (cited 2+ times, blocks downstream synthesis), spawn ONE sub-research targeted at the tension. Max depth 1 (no recursion).
- Sub-research writes to raw/, feeds back into round 3 synthesis.
- Bounded cost: +1 round in worst case, capped budget.
- Mirrors what a careful researcher does manually.

These two compose: `/vault-policy` may itself emit `branch_on_tension: true` for topics where prior vault contradictions exist. So the policy generator becomes the dispatcher for adaptive depth.

### What NOT to add (theater watch)

- **Debate rounds inside personas.** Landscape merge already preserves disagreement. Don't force convergence.
- **Full Tree of Thoughts.** No verifier for research quality. Cost without accuracy gain.
- **N-agent jury for synthesis pages.** `/vault-challenge` already stress-tests; adding a jury duplicates without adding signal.
- **Parallel `/vault-autoresearch` swarm with shared store.** AgentRxiv pattern works for compute-heavy ML training loops, not for human-paced knowledge accumulation. Premature for current vault scale.
- **"Curiosity reward" for self-direction.** RL-style intrinsic motivation needs episodes and reward signal vault doesn't have. `/vault-probe` + `/vault-hypothesize` already cover the productive part.

---

## Recommendation

Build `/vault-policy` (proposal A). It is the highest-ROI addition because:

1. Directly answers user's stated need ("topic-aware quality determination").
2. Single agent, single call, cheap, composable.
3. Becomes the dispatcher that other adaptive features (B above, future autopilot in Q4) plug into.
4. Replaces fixed defaults with adaptive defaults — no new flags for the user to learn, just smarter behavior.

Defer everything else. Specifically:

- Multi-agent debate inside vault is theater given `/vault-landscape` already exists. The 2025 evidence says forcing convergence DELETES correct minority answers; landscape's preserve-disagreement merge is the better pattern.
- ToT-style search is wrong without a verifier; selective deepening via `--branch-on-tension` (proposal B) is the bounded version, but only ship it after `/vault-policy` so it has a dispatcher.
- Autonomous next-pick (`/vault-next`) is a thin wrapper over existing `/vault-probe` + `/vault-hypothesize` + leverage scoring already in `/vault-autoresearch`. Compose, do not invent.

Honest assessment: vault already implements most of the SOTA patterns that matter for research (heterogeneous parallel personas, iterative deepening, adversarial challenge, self-direction via hypothesize/probe). The actual gap is *adaptive routing* — knowing which depth/breadth to apply per topic. That is one planner agent, not a multi-agent system.
